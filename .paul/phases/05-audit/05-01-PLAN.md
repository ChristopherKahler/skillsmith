<objective>
## Goal
Build `/skillsmith audit` — a reusable audit workflow that assesses any skill against the 7 syntax specs and produces a structured compliance report.

## Purpose
Completes the Skillsmith toolbelt (discover → scaffold → distill → audit). Users can audit any skill on demand, not just as a one-off project task. This is the quality enforcement arm of the framework.

## Output
- `skillsmith/tasks/audit.md` — Audit workflow (conversational, spec-driven)
- `skillsmith/skillsmith.md` — Updated with `/skillsmith audit` command and routing
</objective>

<task type="auto">
  <name>Task 1: Create audit workflow</name>
  <files>skillsmith/tasks/audit.md</files>
  <action>
    Create the audit task workflow following the same structural pattern as discover.md and distill.md.

    Structure:
    - purpose: Assess skills against Skillsmith syntax specs, produce compliance report
    - user-story: "As a skill builder, I want to audit existing skills against the syntax standards so I can identify and fix compliance gaps"
    - when-to-use: Auditing a skill after building, checking existing skills for compliance, quality gate before distribution
    - context: @rules/tasks-rules.md (always load audit rules context)
    - references: All 7 specs as on-demand references

    Steps (conversational flow, matching distill.md pattern):

    **Step 1: Identify Target** (priority: first)
    - Ask: Provide skill path, or say "batch" to audit all skills in a directory
    - Single skill: validate path exists, identify entry point file
    - Batch: ask for directory path, discover all skill directories within
    - Wait for response

    **Step 2: Inventory Skill Structure**
    - Read skill directory, catalog what exists:
      - Entry point file? Which one?
      - Subfolders present? (tasks/, templates/, frameworks/, context/, checklists/, rules/)
      - Non-standard folders? (data/, scripts/, utils/, etc.)
      - File count per folder
    - Display inventory table to user
    - Map each folder to its corresponding spec

    **Step 3: Assess Entry Point**
    - Load specs/entry-point.md
    - Check entry point against spec requirements:
      - YAML frontmatter (required fields: name, type, version, category, description)
      - XML sections present (activation, persona, commands, routing, greeting)
      - Placeholder conventions ([square] vs {curly})
      - Routing patterns (always-load, on-command, on-demand with @-references)
    - Score: Compliant / Partial / Non-compliant
    - Record specific violations with line references

    **Step 4: Assess Each Folder Type**
    - For each folder that maps to a spec:
      - Load the corresponding spec (e.g., tasks/ → specs/tasks.md)
      - Read representative files in that folder (all if <5, sample if more)
      - Check against spec requirements (structure, required sections, conventions)
      - Score each: Compliant / Partial / Non-compliant
      - Record violations
    - For non-standard folders (data/, scripts/, utils/):
      - Note as "No spec coverage" — not a violation, just unaudited
    - Skip: node_modules/, output/, any generated directories

    **Step 5: Generate Report**
    - Produce a structured compliance report with:

      **Header:** Skill name, path, tier, audit date

      **Summary Table:**
      | Component | Spec | Status | Issues |
      | Entry point | entry-point.md | Compliant/Partial/Non-compliant | N |
      | tasks/ | tasks.md | ... | N |
      | (each folder) | ... | ... | ... |

      **Overall Score:** % of audited components that are Compliant or Partial

      **Violations Detail:** Per-component list of specific issues with file:line refs

      **Remediation Priorities:** Ordered by impact (entry point issues first, then most-used folders)

      **Non-standard Folders:** Listed as informational (no spec coverage)

    - Ask user where to save report (default: next to skill directory as `{skill-name}-AUDIT.md`)
    - For batch audits: generate individual reports + summary rollup

    **Batch handling:**
    - If batch mode: loop Step 2-5 for each skill discovered
    - After all complete: generate a rollup summary with cross-skill compliance matrix
  </action>

    - Supports both single-skill and batch modes
